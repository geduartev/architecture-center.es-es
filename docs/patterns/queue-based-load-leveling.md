# <a name="queue-based-load-leveling-pattern"></a><span data-ttu-id="5037b-104">Patrón Queue-Based Load Leveling</span><span class="sxs-lookup"><span data-stu-id="5037b-104">Queue-Based Load Leveling pattern</span></span>

[!INCLUDE [header](../_includes/header.md)]

<span data-ttu-id="5037b-105">Use una cola que actúa como búfer entre una tarea y un servicio que invoca para equilibrar cargas pesadas intermitentes que pueden provocar errores en el servicio o la interrupción de la tarea. Esto puede ayudarle a minimizar el impacto de picos de demanda sobre la disponibilidad y capacidad de respuesta de la tarea y el servicio.</span><span class="sxs-lookup"><span data-stu-id="5037b-105">Use a queue that acts as a buffer between a task and a service it invokes in order to smooth intermittent heavy loads that can cause the service to fail or the task to time out. This can help to minimize the impact of peaks in demand on availability and responsiveness for both the task and the service.</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="5037b-106">Contexto y problema</span><span class="sxs-lookup"><span data-stu-id="5037b-106">Context and problem</span></span>

<span data-ttu-id="5037b-107">Muchas soluciones en la nube implican la ejecución de tareas que invocan servicios.</span><span class="sxs-lookup"><span data-stu-id="5037b-107">Many solutions in the cloud involve running tasks that invoke services.</span></span> <span data-ttu-id="5037b-108">En este entorno, si un servicio está vinculado a cargas pesadas intermitentes, puede provocar problemas de confiabilidad o rendimiento.</span><span class="sxs-lookup"><span data-stu-id="5037b-108">In this environment, if a service is subjected to intermittent heavy loads, it can cause performance or reliability issues.</span></span>

<span data-ttu-id="5037b-109">Un servicio puede formar parte de la misma solución que las tareas que lo usan, o puede ser un servicio de terceros que proporciona acceso a los recursos que se usan frecuentemente como una memoria caché o un servicio de almacenamiento.</span><span class="sxs-lookup"><span data-stu-id="5037b-109">A service could be part of the same solution as the tasks that use it, or it could be a third-party service providing access to frequently used resources such as a cache or a storage service.</span></span> <span data-ttu-id="5037b-110">Si el mismo servicio se usa por varias tareas que se ejecutan al mismo tiempo, puede ser difícil predecir el volumen de solicitudes en el servicio en cualquier momento.</span><span class="sxs-lookup"><span data-stu-id="5037b-110">If the same service is used by a number of tasks running concurrently, it can be difficult to predict the volume of requests to the service at any time.</span></span>

<span data-ttu-id="5037b-111">Un servicio puede experimentar picos de demanda que provoquen que se sobrecargue y que no pueda responder a las solicitudes de una manera oportuna.</span><span class="sxs-lookup"><span data-stu-id="5037b-111">A service might experience peaks in demand that cause it to overload and be unable to respond to requests in a timely manner.</span></span> <span data-ttu-id="5037b-112">El desbordamiento de un servicio con muchas solicitudes simultáneas también puede provocar errores en el servicio si no puede controlar la contención que provocan estas solicitudes.</span><span class="sxs-lookup"><span data-stu-id="5037b-112">Flooding a service with a large number of concurrent requests can also result in the service failing if it's unable to handle the contention these requests cause.</span></span>

## <a name="solution"></a><span data-ttu-id="5037b-113">Solución</span><span class="sxs-lookup"><span data-stu-id="5037b-113">Solution</span></span>

<span data-ttu-id="5037b-114">Refactorizar la solución e introducir una cola entre la tarea y el servicio.</span><span class="sxs-lookup"><span data-stu-id="5037b-114">Refactor the solution and introduce a queue between the task and the service.</span></span> <span data-ttu-id="5037b-115">La tarea y el servicio se ejecutan de manera asincrónica.</span><span class="sxs-lookup"><span data-stu-id="5037b-115">The task and the service run asynchronously.</span></span> <span data-ttu-id="5037b-116">La tarea publica un mensaje que contiene los datos que necesita el servicio para una cola.</span><span class="sxs-lookup"><span data-stu-id="5037b-116">The task posts a message containing the data required by the service to a queue.</span></span> <span data-ttu-id="5037b-117">La cola actúa como un búfer, almacenando el mensaje hasta que se recupera mediante el servicio.</span><span class="sxs-lookup"><span data-stu-id="5037b-117">The queue acts as a buffer, storing the message until it's retrieved by the service.</span></span> <span data-ttu-id="5037b-118">El servicio recupera los mensajes de la cola y los procesa.</span><span class="sxs-lookup"><span data-stu-id="5037b-118">The service retrieves the messages from the queue and processes them.</span></span> <span data-ttu-id="5037b-119">Las solicitudes de varias tareas, que pueden generarse a una frecuencia muy variable, pueden pasarse al servicio mediante la misma cola de mensajes.</span><span class="sxs-lookup"><span data-stu-id="5037b-119">Requests from a number of tasks, which can be generated at a highly variable rate, can be passed to the service through the same message queue.</span></span> <span data-ttu-id="5037b-120">En esta figura se muestra el uso de una cola para nivelar la carga de un servicio.</span><span class="sxs-lookup"><span data-stu-id="5037b-120">This figure shows using a queue to level the load on a service.</span></span>

![Figura 1: uso de una cola para nivelar la cargar de un servicio](./_images/queue-based-load-leveling-pattern.png)

<span data-ttu-id="5037b-122">La cola desvincula las tareas del servicio, y el servicio puede controlar los mensajes a su propio ritmo independientemente del volumen de solicitudes de las tareas simultáneas.</span><span class="sxs-lookup"><span data-stu-id="5037b-122">The queue decouples the tasks from the service, and the service can handle the messages at its own pace regardless of the volume of requests from concurrent tasks.</span></span> <span data-ttu-id="5037b-123">Además, no existe ningún retraso en una tarea si el servicio no está disponible en el momento en que publica un mensaje en la cola.</span><span class="sxs-lookup"><span data-stu-id="5037b-123">Additionally, there's no delay to a task if the service isn't available at the time it posts a message to the queue.</span></span>

<span data-ttu-id="5037b-124">Este modelo proporciona las siguientes ventajas:</span><span class="sxs-lookup"><span data-stu-id="5037b-124">This pattern provides the following benefits:</span></span>

- <span data-ttu-id="5037b-125">Puede ayudar a maximizar la disponibilidad porque los retrasos que surgen de los servicios no tendrán un impacto directo e inmediato sobre la aplicación, que puede seguir publicando mensajes a la cola incluso cuando el servicio no está disponible o no está procesando mensajes en esos momentos.</span><span class="sxs-lookup"><span data-stu-id="5037b-125">It can help to maximize availability because delays arising in services won't have an immediate and direct impact on the application, which can continue to post messages to the queue even when the service isn't available or isn't currently processing messages.</span></span>
- <span data-ttu-id="5037b-126">Puede ayudar a maximizar la escalabilidad porque el número de colas y el número de servicios puede variar para satisfacer la demanda.</span><span class="sxs-lookup"><span data-stu-id="5037b-126">It can help to maximize scalability because both the number of queues and the number of services can be varied to meet demand.</span></span>
- <span data-ttu-id="5037b-127">Puede ayudar a controlar los costos porque el número de instancias de servicio que se ha implementado solo tiene que ser adecuado para cumplir la carga media en lugar de la carga máxima.</span><span class="sxs-lookup"><span data-stu-id="5037b-127">It can help to control costs because the number of service instances deployed only have to be adequate to meet average load rather than the peak load.</span></span>

    >  <span data-ttu-id="5037b-128">Algunos servicios implementan limitaciones cuando la demanda alcanza un umbral en el que el sistema puede producir errores.</span><span class="sxs-lookup"><span data-stu-id="5037b-128">Some services implement throttling when demand reaches a threshold beyond which the system could fail.</span></span> <span data-ttu-id="5037b-129">La limitación puede reducir la funcionalidad disponible.</span><span class="sxs-lookup"><span data-stu-id="5037b-129">Throttling can reduce the functionality available.</span></span> <span data-ttu-id="5037b-130">Puede implementar una redistribución de la carga con estos servicios para garantizar que este umbral no se alcance.</span><span class="sxs-lookup"><span data-stu-id="5037b-130">You can implement load leveling with these services to ensure that this threshold isn't reached.</span></span>
